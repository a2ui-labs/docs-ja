# コンセプト概要

A2UIは、AIエージェントが任意のコードを実行することなく、リッチでインタラクティブなUIを生成できるように設計されています。これを実現するために、いくつかの主要なコンセプトに基づいています。

主要な用語の簡単な説明については、[用語集](glossary.md) を参照してください。

## 3つのコア要素

A2UIのメッセージは、通常以下の3つの要素を分離して扱います：

1.  **UI構造 (Structure)**: どのコンポーネントをどこに配置するかを定義します。
2.  **アプリケーションの状態 (State / Data Model)**: UIに表示される実際のデータを保持します。
3.  **レンダリング開始 (Signals)**: UIを表示する準備が整ったことをクライアントに伝えます。

これらを分離することで、エージェントはデータのみを更新したり、UIの構造を段階的に構築したりすることができ、ストリーミングに最適化された柔軟な対話が可能になります。

## 主要なコンセプト

### 1. サーフェス (Surface)
[サーフェス](overview.md#surface)は、UIがレンダリングされる独立した「キャンバス」です。ダイアログ、サイドバー、メインチャットウィンドウなど、アプリ内の特定の領域を指します。

### 2. コンポーネント (Component)
[コンポーネント](components.md)は、A2UIの基本的なビルディングブロックです。`Button`, `Text`, `TextField`, `Card` などの標準的なUI要素が含まれます。

### 3. データバインディング (Data Binding)
[データバインディング](data-binding.md)は、コンポーネントのプロパティをデータモデル内の特定の値に紐付けます。これにより、データが更新されるとUIが自動的に反映されます。

### 4. データフロー (Data Flow)
[データフロー](data-flow.md)は、エージェントからクライアントへメッセージがどのように流れ、ユーザーのアクションがどのようにエージェントにフィードバックされるかを定義します。

## メッセージタイプ

=== "v0.9(安定版)"

    v0.9 では、次のメッセージタイプを使用します。

    - **`createSurface`**: 新しい Surface を作成し、そのカタログを指定します
    - **`updateComponents`**: Surface 内の UI コンポーネントを追加または更新します
    - **`updateDataModel`**: アプリケーションの状態を更新します
    - **`deleteSurface`**: UI Surface を削除します

    v0.9 では、Surface の作成とレンダリングが分離されました。`createSurface` が `beginRendering` と `surfaceUpdate` による暗黙的な Surface 作成の両方を置き換えます。すべてのメッセージに `version` フィールドが含まれます。

=== "v1.0(候補版)"

    v1.0 では、次のメッセージタイプを使用します。

    - **`createSurface`**: 新しい Surface を作成し、そのカタログを指定します
    - **`updateComponents`**: Surface 内の UI コンポーネントを追加または更新します
    - **`updateDataModel`**: アプリケーションの状態を更新します
    - **`deleteSurface`**: UI Surface を削除します
    - **`actionResponse`**: クライアント起点のアクションに応答します

    v1.0 では `actionResponse` メッセージタイプが導入され、堅牢なクライアント-サーバー間の同期 RPC 機能が可能になります。

=== "v0.8(レガシー)"

    v0.8 では、次のメッセージタイプを使用します。

    - **`surfaceUpdate`**: UI コンポーネントを定義または更新します
    - **`dataModelUpdate`**: アプリケーションの状態を更新します
    - **`beginRendering`**: クライアントにレンダリングを指示します
    - **`deleteSurface`**: UI Surface を削除します

詳しい技術仕様については、[メッセージリファレンス](../reference/messages.md) を参照してください。
