# レンダラー (クライアントライブラリ)

レンダラーは、A2UI JSONメッセージを各プラットフォーム用のネイティブUIコンポーネントに変換します。

[エージェント](agents.md)は A2UI メッセージの生成を担当し、[トランスポート層](transports.md)はメッセージをクライアントに配信する役割を担います。クライアントのレンダラーライブラリは、A2UIメッセージをバッファリングして処理し、A2UIのライフサイクルを実装し、サーフェス（ウィジェット）をレンダリングする必要があります。

レンダラーにカスタムコンポーネントを追加したり、独自のUIコンポーネントフレームワークをサポートするために独自のレンダラーを構築したりすることもできる柔軟性を提供しています。

詳細は[ロードマップ](roadmap.md)を確認してください。

## レンダラーの仕組み

```
A2UI JSON → レンダラー → ネイティブコンポーネント → あなたのアプリ
```

1. トランスポート層からA2UIメッセージを**受信**
2. JSONの**パース**とスキーマの検証
3. プラットフォームのネイティブコンポーネントを使用した**レンダリング**
4. アプリのテーマに合わせた**スタイリング**

## レンダラーを構築する

独自のプラットフォーム用レンダラーを作成したいですか？

- 計画中のフレームワークについては、[ロードマップ](roadmap.md)を確認してください。
- 既存のレンダラーのパターンを検討してください。
- レンダラー実装の詳細については、[レンダラー開発ガイド](guides/renderer-development.md)を確認してください。

### 主な要件：

- A2UI JSONメッセージ、特に隣接リスト (adjacency list) 形式のパース
- A2UIコンポーネントのネイティブウィジェットへのマッピング
- データバインディングとライフサイクルイベントの処理
- UIを構築・更新するための、順次的な増分A2UIメッセージ処理
- サーバー主導のアップデートのサポート
- ユーザーアクションのサポート
